# v2 review guide

Use this checklist while reviewing the prototype:

1. Start the workout and confirm the persistent timing ribbon behaves as expected.
2. On Workout, select exercises, drag them into a new order, collapse exercises, hide completed sets, and use **Complete + rest**.
3. Enter workout-level notes and exercise-level notes; confirm parsed tags are useful.
4. On Program, drag days, edit a day title/status/notes, add exercises, then drag exercises into the desired order.
5. On Progression, toggle **Skip progression for this cycle**, edit a recommendation, accept it as default, and verify the log captures the action.
6. On Analytics, change lookback and tracked exercise.
7. On Devices, simulate and resolve a conflict.
8. On Settings, test density, buttons, accent color, and exercise database additions.
9. Use the Feedback button to save screen-specific notes and export JSON for Codex.

The prototype intentionally uses static/mock data. The goal is interaction and workflow validation before implementation.
